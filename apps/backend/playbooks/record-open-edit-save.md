# Detail Open Edit Commit

- Treat lists and detail views as separate contexts.
- When the user names a record, select/open that record before editing any field or adding any note/comment.
- Verify record identity using strong visible anchors such as title, issue key, customer name, contact name, or detail header.
- Do not edit inline list cells when the user intent is clearly about the full record.
- Before save/submit/close actions, verify that the edited content belongs to the intended record and not a previously active one.
- If multiple records share similar labels, prefer exact matches plus contextual metadata rather than partial text alone.

# Editor Draft And Commit

- Break the flow into three phases: select context, draft content, then commit.
- Never type into a downstream editor until the intended context is active.
- Distinguish lookup/search fields from actual editor fields.
- Before clicking send, save, submit, publish, or post, verify both the active context and the drafted content.
- If the UI still shows the old context while presenting new search results or suggestions, treat that as unresolved selection.
- For follow-up edits, reuse the current context only when the visible header, title, or detail state still matches the intended target.
